設定ファイルの書き方
-------------------------------------------------
  設定ファイルは1行が1つの設定に対応しており、
  それぞれの行は以下のフォーマットで書き込まれます。

    設定名 = 設定値

  設定名および設定値の前後の空白は全て無視されます。

  そしてこれらの設定はさらにセクションごとに別々に管理されます。
  "["と"]"と囲まれた行がセクションの開始を示します。

  これはWindowsのINIファイルと同じフォーマットです。


ボタン定義について
-------------------------------------------------
  設定値の内容は、設定によって様々ですが、
  単なる数値やON/OFF以外によく使っているものとして、ボタン定義があります。
  ボタン定義は、ボタンの組み合わせを設定する場合に使用されます。

  それぞれのボタンは以下の表の名前と対応しており、
  これらの名前をひとつ、あるいは「+」でつないで複数列挙します。
  大文字小文字は区別しません。

      ------------------------
      ボタン名   | ボタン
      ------------------------
      Select     | セレクトボタン
      Start      | スタートボタン
      Up         | 十字キー上
      Right      | 十字キー右
      Down       | 十字キー下
      Left       | 十字キー左
      LTrigger   | Lボタン
      RTrigger   | Rボタン
      Triangle   | △ボタン
      Circle     | ○ボタン
      Cross      | ×ボタン
      Square     | □ボタン
      HOME       | HOMEボタン
      NOTE       | ♪ボタン
      SCREEN     | 輝度変更ボタン
      VolUp      | 音量+ボタン
      VolDown    | 音量-ボタン
      AnalogUp   | アナログスティック上方向
      AnalogRight| アナログスティック右方向
      AnalogDown | アナログスティック下方向
      AnalogLeft | アナログスティック左方向
      RPlayPause | リモコンの再生/一時停止ボタン
      RForward   | リモコンの早送りボタン
      RBack      | リモコンの巻き戻しボタン
      RVolUp     | リモコンの音量+ボタン
      RVolDown   | リモコンの音量-ボタン
      ------------------------

  たとえば、スタートボタンとセレクトボタンの同時押しにする場合は

      Start + Select

  のように記述します。
  また、単にボタン一つだけ、たとえば♪ボタンだけを設定する場合には、

      NOTE

  のように、単純に名前だけを記述します。

  ちなみに、いないとは思いますが、

      Left + Right
      Up + Down
      Up + Right + Down + Left

  にような設定も可能です。
  ですが、通常PSPの十字ボタンは押した方向と逆側のボタンを同時に押すことはできませんので、
  これは意味をなしません。

  それどころか、このような設定をメニュー起動ボタンにしてしまうと、
  その間メニューを表示できなくなりますのでご注意ください。
  MacroFireは、このようなあり得ない設定値を修正しません。


HOMEボタンについて
-------------------------------------------------
  HOMEボタンにはPSPの仕様による制約をいくつか受けます。
  詳しくはreadme.txtの「HOMEボタンの制約」の項を参照してください。


設定
-------------------------------------------------
  設定可能な設定名と設定値、およびその詳細を紹介します。

  [Main]セクション
  -----------------------------------------------
    Startup
      設定値: ON | OFF

      起動時にMacroFire Engineを有効にするかどうかを設定します。
        OFF : 起動時に有効状態にしない。
        ON  : 起動時に有効状態にする。

    MenuButtons
      設定値: ボタン定義

      デフォルトのButtons to launch the menuを設定します。
      詳細はreadme.txt参照。

    ToggleButtons
      設定値: ボタン定義

      デフォルトのButtons to toggle the engine stateを設定します。
      詳細はreadme.txt参照。

    StatusNotification
      設定値: ON | OFF

      デフォルトのEngine status notificationを設定します。
      詳細はreadme.txt参照。

  [AlternativeButtons]セクション
  -----------------------------------------------
    Select
    Start
    Up
    Right
    Down
    Left
    Ltrigger
    Rtrigger
    Triangle
    Circle
    Cross
    Square
    Home
      設定値: ボタン定義

      MacroFireのメニューの中でのみ機能する代替ボタンの設定です。
      壊れたボタンをカバーするのが主な目的です。

      上記のボタンに対して代替ボタンを設定します。
      設定値が空の場合は代替ボタンは設定されず、そのままボタン名通りのボタンを使います。

      例えば、スタートボタンが壊れていて反応しないなどの理由により
      スタートボタンを代替ボタンを設定したい場合は、
      Startエントリにのみ、代替ボタンにしたいボタンの組み合わせを設定します。

        Start = LTrigger + RTrigger + Select

  [AnalogStick]セクション
  -----------------------------------------------
    Movement
      設定値: ON | OFF

      デフォルトのAnalog stick movementを設定します。
      詳細はmanual_analogsens.txt参照。

    Deadzone
      設定値: 0-182の数値

      デフォルトのDeadzone radiusを設定します。
      詳細はmanual_analogsens.txt参照。

    Sensitivity
      設定値: 0-200の数値

      デフォルトのSensitivityを設定します。
      詳細はmanual_analogsens.txt参照。

    OriginX
      設定値: 0-255の数値

      デフォルトのOrigin X-coordinateを設定します。
      詳細はmanual_analogsens.txt参照。

    OriginY
      設定値: 0-255の数値

      デフォルトのOrigin Y-coordinateを設定します。
      詳細はmanual_analogsens.txt参照。


  [Default]セクション
  -----------------------------------------------
    UseGraphicsMemory
      設定値: ON | OFF

      ユーザメモリに空きがある場合、メニュー表示用にメモリを確保することを許可します。
      安全に使用できるはずのメモリを確保しているので、ほとんどの場合ONでも問題ないはずです。
      ところが、希にメニューを表示した時、あるいは終了した時にフリーズするソフトがあるようで、
      その場合はこれをOFFにすると解決する可能性があります。
      ただし、メニュー上での操作性は低下します。

    UseVolatileMemory
      設定値: ON | OFF

      UseGraphicsMemoryがONのとき、使用するメモリをユーザメモリではなく4MBのカーネルメモリを使うかどうかを設定します。
      ユーザメモリが少ない状態でもメニューの速度を低下させずに操作できるようになります。
      ただし、ゲーム側がこの領域のメモリを使用している場合はおそらくフリーズします。
      ネットワークに接続するゲームではONにしないほうが無難です。

    Remap
      設定値: ファイルパス(ms0:/～)

      起動時に読み込ませるボタン再割り当て設定ファイルを指定します。
      「ms0:/」からのフルパスで記述してください。

    Rapidfire
      設定値: ファイルパス(ms0:/～)

      起動時に読み込ませる連射設定ファイルを指定します。
      「ms0:/」からのフルパスで記述してください。

    Macro1
    Macro2
    Macro3
    Macro4
    Macro5
    Macro6
    Macro7
    Macro8
    Macro9
    Macro10
      設定値: ファイルパス(ms0:/～)

      起動時に読み込ませるマクロファイルを指定します。
      「ms0:/」からのフルパスで記述してください。


起動時に読み込まれる設定ファイルをゲームごとに切り変えるには
-------------------------------------------------
  macrofire.iniにゲームIDのセクション名を記入し、そこに設定を書き込みます。
  たとえば以下のように:
  
      [ULJM-05325]
      UseGraphicsMemory = On
      UseVolatileMemory = Off
      Remap     = ms0:/macrofire/remap/analog_to_dpad.ini
      Rapidfire = ms0:/macrofire/rapidfire/SOSE.ini
      Macro1    = ms0:/macrofire/macro/SOSE/auto_creation_shippitsu.ini
      Macro2    = ms0:/macrofire/macro/SOSE/auto_creation_art.ini
  
  このように設定することで、ゲームIDがULJM-05325のゲームを起動したときのみ
  設定されているファイルを読み込みます。
  
  このゲームIDは、ゲームを起動した状態でMacroFireのメニューを表示させると、
  最上部に表示されています。
  
  必要のないエントリは省略できます。
  この場合、Macro3～10は使用していないので、書いていません。

  書き込めるエントリは、[Default]セクションと同じです。


MacroFireメニュー上部の[ID: xxx / Loaded: xxx]の意味
-------------------------------------------------
  MacroFireのメニューを表示させると、最上部に
  
      [ID: xxx / Loaded: xxx]
  
  あるいは、単純に
  
      [ID: xxx]
  
  と表示される場合があります。
  
  これは、MacroFireが読み込もうとした設定ファイルのセクション名と、
  実際に読み込んだセクション名を表しています。

  この双方が一致すれば単純にIDのみを表示しますが、
  読み込もうとしたIDが存在せず、別のセクションを読み込んだ場合は「ロード」部分で実際に読み込んだセクション名を表示します。
  
  例えば、

      ゲームが起動され[ULJM-05325]というセクションを読み込もうとしたが、
      そのセクションがINIファイルに設定されていないため、最終的に[Default]セクションを読み込んだ

  という場合は、

      [ID: ULJM-05325 / Loaded: Default]

  と表示されます。

  逆に、[ULJM-05325]を読み込もうとし、それが成功した場合は、

      [ID: ULJM-05325]

  と表示されます。

  このように、MacroFireは目的のセクションが見つからない場合、
  何段階かに分かれて別のセクションを探します。

  また、目的のセクションが見つかった場合は、それより下位のセクションは読みません。
  例えば目的のセクションが[ULJM-05325]だった時、

      [ULJM-05325]セクションのMacro1-10には何も設定されていないが、
      [Default]セクションにはMacro1-5が設定されている

  という場合、Macro1-10すべてに何も設定されません。
  [ULMJ-05325]セクションのMacro1-5が空だからといって、
  それを[Default]セクションから引っ張ってきたりはしません。


[ID: xxx / Loaded: Built-in]について
-------------------------------------------------
  前述の表示が Loaded: Built-in と表示されている場合、これはmacrofire.iniの読み込みに失敗しています。

  macrofire.iniの読み込みに失敗し、更に新規作成も失敗した場合に
  内部で設定されている初期設定値を使用します。

  Built-in表示は、その初期設定値を使用していることを示します。

  Built-inというセクションを読んでいるわけではないので、
  macrofire.iniに[Built-in]というセクションを作ってもそこは使われません。
  そもそも、これが表示されている時はmacrofire.iniを読み込めていないので、セクションがどうのという問題ではありません。


起動時に検索されるセクションの名前と順番
-------------------------------------------------
  XMB:
    [VSH] -> [Default]

  POPS (今のところPOPS上でPSゲームのゲームIDを取得できない):
    [POPS] -> [Default]

  GAME:
    [<ゲームID (あれば)>] -> [GAME] -> [Default]
